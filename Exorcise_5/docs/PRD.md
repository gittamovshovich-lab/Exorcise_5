# PRD — Product Requirements Document
## מערכת יצירת מאמרים מקצועיים באמצעות CrewAI

**גרסה:** 1.0  
**תאריך:** 2026-06-16  
**סטטוס:** Approved  

---

## 1. סקירה כללית (Overview)

### 1.1 מטרת המוצר

לבנות מערכת אוטומטית המייצרת מאמרים מקצועיים מאיכות גבוהה על כל נושא נתון,
באמצעות pipeline של 4 סוכני AI המתמחים כל אחד בשלב אחד של תהליך כתיבת תוכן.

### 1.2 הגדרת הבעיה

יצירת תוכן מקצועי דורשת שילוב של מיומנויות שונות: מחקר, כתיבה, עריכה ועיצוב.
בדרך כלל, אדם אחד אינו מצטיין בכולן, ועלות העסקת מומחים נפרדים גבוהה.
מערכת Multi-Agent יכולה לחקות תהליך ייצור תוכן מקצועי על ידי הקצאת "עובד AI" מתמחה לכל שלב.

### 1.3 קהל היעד

- **ראשוני:** מגישי המטלה האקדמית (הדגמת הבנת Multi-Agent Orchestration)
- **שניוני:** ארגונים המעוניינים לאוטומט ייצור תוכן

## 2. דרישות פונקציונליות (Functional Requirements)

### FR-01: קבלת קלט
- המערכת חייבת לקבל נושא מאמר כ-string
- ניתן להעביר הנושא דרך command-line argument (`--topic`)
- ישנו נושא ברירת מחדל ("The Impact of AI on Modern Healthcare")

### FR-02: שלב מחקר
- Research Agent חייב לבצע חיפוש אינטרנטי עצמאי
- הפלט חייב לכלול: עובדות, סטטיסטיקות, דעות מומחים, מגמות עדכניות, ומקורות
- פורמט הפלט: תקציר מחקר מובנה עם כותרות (500-700 מילים)

### FR-03: שלב כתיבה
- Writer Agent חייב לבסס את המאמר **אך ורק** על ממצאי המחקר
- מבנה חובה: כותרת, הקדמה, 3-4 פרקי גוף עם כותרות-משנה, מסקנה
- אורך: 800-1000 מילים

### FR-04: שלב עריכה
- Editor Agent חייב לבצע 6 בדיקות: דיוק עובדתי, מבנה, בהירות, סגנון, דקדוק, עקביות
- הפלט חייב להיות המאמר **המלא** הערוך, לא רק רשימת תיקונים
- חובה לכלול הערת עורך קצרה

### FR-05: שלב פרסום
- Publisher Agent חייב להוסיף: Metadata block, תיאור SEO, תגים, תקציר מנהלים, Markdown hierarchy, CTA, נושאים קשורים
- הפלט חייב להיות מסמך Markdown תקני מוכן לפרסום

### FR-06: שמירת פלט
- הפלט הסופי חייב להישמר אוטומטית לקובץ `output_article.md`

### FR-07: שקיפות תהליך
- המערכת חייבת לפעול עם `verbose=True`
- כל שלב חייב להיות מודפס לקונסול בזמן אמת

## 3. דרישות לא-פונקציונליות (Non-Functional Requirements)

### NFR-01: סדרתיות
- **חובה** שהתהליך יהיה `Process.sequential`
- אסור להריץ סוכנים במקביל (לא `Process.hierarchical` ולא `Process.parallel`)

### NFR-02: העברת context
- כל Task חייבת לקבל את הפלט של הקודמת דרך `context=[previous_task]`
- אסור להעביר מידע דרך משתנים גלובליים או קבצים חיצוניים

### NFR-03: עצמאות סוכנים
- כל סוכן חייב להיות מוגדר עם `allow_delegation=False`
- אסור לסוכן לפנות לסוכן אחר ישירות

### NFR-04: מבנה קוד
- הגדרות Agents, Tasks, Crew וה-entry point חייבים להיות בקובץ יחיד (`main.py`)
- קוד נקי עם comments בעברית/אנגלית

### NFR-05: ניהול סודות
- API keys חייבים להיטען מקובץ `.env` ולא להיות hard-coded בקוד

## 4. הגדרות Agents (Agent Specifications)

| Agent             | Role                          | Goal                                      | Tools           |
|-------------------|-------------------------------|-------------------------------------------|-----------------|
| research_agent    | Senior Research Analyst       | לאסוף מידע מקיף ומאורגן                  | SerperDevTool   |
| writer_agent      | Professional Content Writer   | להפוך מחקר למאמר מרתק                    | ללא             |
| editor_agent      | Senior Content Editor         | לשפר ולסנן לרמה מקצועית                  | ללא             |
| publisher_agent   | Content Publisher & Formatter | לעצב לפרסום מוכן                         | ללא             |

## 5. הגדרות Tasks

| Task              | Agent            | Context מ-          | Expected Output                          |
|-------------------|------------------|---------------------|------------------------------------------|
| research_task     | research_agent   | (קלט ראשוני)        | תקציר מחקר מובנה 500-700 מילים          |
| writing_task      | writer_agent     | research_task       | מאמר מלא 800-1000 מילים                 |
| editing_task      | editor_agent     | writing_task        | מאמר ערוך + הערת עורך                    |
| publishing_task   | publisher_agent  | editing_task        | מסמך Markdown מלא מוכן לפרסום          |

## 6. מגבלות וסייגים (Constraints)

- **LLM:** מבוסס על OpenAI (GPT-4o כברירת מחדל) — CrewAI אינו מגביל למודל ספציפי
- **כלי חיפוש:** SerperDevTool דורש SERPER_API_KEY (חינמי עד 2,500 בקשות/חודש)
- **עלות:** כל הרצה מייצרת עלות API — מומלץ לבדוק בסביבת sandbox תחילה
- **שפה:** הקוד באנגלית; התיעוד דו-לשוני

---

## 7. קריטריוני הצלחה (Success Criteria)

- [ ] המערכת רצה ללא שגיאות על נושא כלשהו
- [ ] ה-pipeline מבצע את כל 4 השלבים בסדר הנכון
- [ ] הפלט הסופי הוא מסמך Markdown קריא ומקצועי
- [ ] context עובר בין כל הסוכנים (מוכח ע"י verbose output)
- [ ] הקוד עומד בכל הדרישות הטכניות של המטלה האקדמית
